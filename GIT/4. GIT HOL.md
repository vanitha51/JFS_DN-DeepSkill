**Git Hands-on Lab – Merge Conflict Resolution**



Step 1: Verify if Master is in Clean State

Command

git status

Output

On branch master



nothing to commit, working tree clean

Observation



The master branch is clean and ready for the lab.

\----------------------------------------------------

**Step 2:** Create a New Branch "GitWork"

Command

git branch GitWork

Output

(No output)

Observation



The branch GitWork was created successfully.

\----------------------------------------------------

**Step 3:** Switch to GitWork Branch

Command

git checkout GitWork

Output

Switched to branch 'GitWork'

Observation



The current branch is now GitWork.

\----------------------------------------------------

**Step 4:** Create hello.xml

Command

echo "<message>Hello from GitWork Branch</message>" > hello.xml

Output

(No output)

Observation



The file hello.xml was created.

\----------------------------------------------------

**Step 5:** Check Git Status

Command

git status

Output

On branch GitWork



Untracked files:

&#x20;   hello.xml

Observation



Git detects the newly created file.

\----------------------------------------------------

**Step 6:** Stage the File

Command

git add hello.xml

Output

(No output)

Observation



The file was added to the staging area.

\----------------------------------------------------

**Step 7:** Commit the File

Command

git commit -m "Added hello.xml in GitWork"

Output

\[GitWork abc1234] Added hello.xml in GitWork

Observation



The file was committed to GitWork.

\----------------------------------------------------

**Step 8:** Update hello.xml

Command

echo "<message>Hello from GitWork Branch - Updated</message>" > hello.xml

Output

(No output)

Observation



The content of hello.xml was modified.

\----------------------------------------------------

**Step 9**: Check Git Status

Command

git status

Output

modified: hello.xml

Observation



Git identifies the file as modified.

\----------------------------------------------------

**Step 10:** Commit the Updated File

Command

git add hello.xml



git commit -m "Updated hello.xml in GitWork"

Output

\[GitWork def5678] Updated hello.xml in GitWork

Observation



The updated version was committed successfully.

\----------------------------------------------------

**Step 11:** Switch Back to Master

Command

git checkout master

Output

Switched to branch 'master'

Observation



Returned to the master branch.

\----------------------------------------------------

**Step 12:** Create hello.xml in Master

Command

echo "<message>Hello from Master Branch</message>" > hello.xml

Output

(No output)

Observation



A different version of hello.xml was created in the master branch.

\----------------------------------------------------

**Step 13:** Commit the Master Changes

Command

git add hello.xml



git commit -m "Added hello.xml in master"

Output

\[master xyz9876] Added hello.xml in master

Observation



The master branch now has a different version of the same file.

\----------------------------------------------------

**Step 14:** View Commit History

Command

git log --oneline --graph --decorate --all

Example Output

\* xyz9876 (HEAD -> master) Added hello.xml in master

| \* def5678 (GitWork) Updated hello.xml in GitWork

| \* abc1234 Added hello.xml in GitWork

|/

\* 1234567 Initial Commit

Observation



The commit graph shows both branches diverging.

\----------------------------------------------------

**Step 15:** View Differences Using Git Diff

Command

git diff master GitWork

Output

Displays line-by-line differences between both branches.

Observation



Git displays the differences in the command line.

\----------------------------------------------------

**Step 16:** View Differences Using P4Merge

Command

git difftool master GitWork

Output

Launching P4Merge...

Observation



P4Merge displays the visual comparison of both versions.

\----------------------------------------------------

**Step 17:** Merge GitWork into Master

Command

git merge GitWork

Output

Auto-merging hello.xml



CONFLICT (add/add): Merge conflict in hello.xml



Automatic merge failed; fix conflicts and then commit the result.

Observation



A merge conflict occurred because hello.xml exists with different content in both branches.

\----------------------------------------------------

**Step 18:** Observe Git Markup

Command

cat hello.xml

Output

<<<<<<< HEAD

<message>Hello from Master Branch</message>

=======

<message>Hello from GitWork Branch - Updated</message>

>>>>>>> GitWork

Observation



Git inserted conflict markers:



<<<<<<< HEAD

=======

>>>>>>> GitWork



These indicate the conflicting sections.

\----------------------------------------------------

**Step 19:** Resolve Conflict Using 3-Way Merge (P4Merge)

Command

git mergetool

Output

Launching P4Merge...

Observation



P4Merge opens the 3-way merge window.



Select the required content or combine both versions.



Save the file and close P4Merge.

\----------------------------------------------------

**Step 20:** Verify Conflict Resolution

Command

git status

Output

All conflicts fixed but you are still merging.

Observation



The conflict has been resolved but must still be committed.

\----------------------------------------------------

**Step 21:** Commit the Merge

Command

git add hello.xml



git commit -m "Resolved merge conflict using P4Merge"

Output

\[master 456abcd] Resolved merge conflict using P4Merge

Observation



The merge conflict has been resolved successfully.

\----------------------------------------------------

**Step 22:** Check Repository Status

Command

git status

Output

On branch master



nothing to commit, working tree clean

Observation



The repository is clean.

\----------------------------------------------------

**Step 23:** Ignore Backup Files



Open .gitignore



Command

notepad .gitignore



Add



\*.bak



Save and close.



Observation



Git will now ignore backup files generated by merge tools.

\----------------------------------------------------

**Step 24:** Commit .gitignore

Command

git add .gitignore



git commit -m "Updated .gitignore to ignore backup files"

Output

\[master 789efgh] Updated .gitignore

Observation



The .gitignore file has been updated successfully.

\----------------------------------------------------

**Step 25:** List Available Branches

Command

git branch

Output

GitWork

\* master

Observation



Both branches are listed.

\----------------------------------------------------

**Step 26:** Delete GitWork Branch

Command

git branch -d GitWork

Output

Deleted branch GitWork.

Observation



The merged branch was deleted successfully.

\----------------------------------------------------

**Step 27:** Verify Remaining Branches

Command

git branch

Output

\* master

Observation



Only the master branch remains.
