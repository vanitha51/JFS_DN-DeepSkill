###### **Git Hands-on Lab – Using .gitignore**



**Step 1:** Open Git Bash and Navigate to Repository

Command

cd /c/Users/YourName/Desktop/GitLab

Output

Current directory changed to GitLab repository.

Observation



Successfully navigated to the Git working directory.

\----------------------------------------------------

**Step 2:** Check the Current Repository Status

Command

git status

Output

On branch master

nothing to commit, working tree clean

Observation



The repository is clean before creating new files.

\----------------------------------------------------

**Step 3:** Create a Log Folder

Command

mkdir log

Output

(No output)

Observation



A folder named log was created successfully.

\----------------------------------------------------

**Step 4:** Create a .log File

Command

echo "Git Ignore Demo" > sample.log

Output

(No output)

Observation



A file named sample.log was created.

\----------------------------------------------------

**Step 5:** Verify the Created Files

Command

ls

Output

log

sample.log

Observation



The working directory now contains the log folder and sample.log file.

\----------------------------------------------------

**Step 6:** Check Git Status

Command

git status

Output

On branch master



Untracked files:

&#x20;   log/

&#x20;   sample.log

Observation



Git detects the newly created folder and log file as untracked.

\----------------------------------------------------

**Step 7:** Create a .gitignore File

Command

touch .gitignore

Output

(No output)

Observation



A .gitignore file was created.

\----------------------------------------------------

**Step 8:** Open .gitignore

Command

notepad .gitignore

Output

Notepad opens.

Observation



The .gitignore file is opened for editing.

\----------------------------------------------------

**Step 9:** Add Ignore Rules

Add the following lines to the .gitignore file:

\*.log

log/



Save the file and close Notepad.



Observation

\*.log ignores all files having the .log extension.

log/ ignores the entire log directory.

\----------------------------------------------------

**Step 10:** Check Git Status Again

Command

git status

Output

On branch master



Untracked files:

&#x20;   .gitignore



nothing added to commit but untracked files present

Observation

sample.log is no longer displayed.

log folder is no longer displayed.

Only .gitignore appears because it is not ignored.

\----------------------------------------------------

**Step 11:** Add .gitignore to the Staging Area

Command

git add .gitignore

Output

(No output)

Observation



The .gitignore file is staged for commit.

\----------------------------------------------------

**Step 12:** Verify Git Status

Command

git status

Output

On branch master



Changes to be committed:

&#x20;   new file: .gitignore

Observation



Only the .gitignore file is staged. The ignored files remain hidden.

\----------------------------------------------------

**Step 13:** Commit the .gitignore File

Command

git commit -m "Added .gitignore file"

Output

\[master abc1234] Added .gitignore file



1 file changed, 2 insertions(+)



create mode 100644 .gitignore

Observation



The .gitignore file has been committed successfully.

\----------------------------------------------------

**Step 14:** Verify Repository Status

Command

git status

Output

On branch master



nothing to commit, working tree clean

Observation



The repository is clean.



The sample.log file and log folder are still present in the working directory but are ignored by Git.

\----------------------------------------------------

**Step 15:** Verify Ignored Files

Command

git status --ignored

Output

On branch master



Ignored files:

&#x20;   log/

&#x20;   sample.log



nothing to commit, working tree clean

Observation



Git confirms that sample.log and the log folder are being ignored according to the rules in .gitignore.
