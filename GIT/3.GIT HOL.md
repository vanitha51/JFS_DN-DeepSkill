**Hands-on Lab: Git Branching and Merging**



**Step 1**:Open Git Bash and Navigate to Repository



Command:

cd /c/Users/YourName/Desktop/GitLab



Output:

Current directory changed to GitLab repository.

Observation:

Successfully navigated to the Git repository.



\----------------------------------------------------



**Step 2**: Check Current Branch



Command:

git branch -a



Output:

\* master

Observation:

The current active branch is master.

\----------------------------------------------------



**Step 3**: Create a New Branch



Command:

git branch GitNewBranch



Output:

(No output if branch is created successfully)



Observation:

A new branch named GitNewBranch was created successfully.

\----------------------------------------------------

**Step 4:** List All Local and Remote Branches



Command:

git branch -a



Output:

GitNewBranch

\* master

remotes/origin/master



Observation:

Local branches displayed.

Remote branches displayed.

The '\*' symbol indicates the currently active branch (master).

\----------------------------------------------------



**Step 5:** Switch to the New Branch



Command:

git checkout GitNewBranch



Output:

git checkout GitNewBranch



Observation:

Successfully switched from master to GitNewBranch.

\----------------------------------------------------



**Step 6:** Verify Current Branch



Command:

git branch



Output:



\* GitNewBranch

master



Observation:

The current active branch is GitNewBranch.

\----------------------------------------------------

**Step 7:** Create a New File



Command:

echo "This file is created in GitNewBranch." > demo.txt



Output:

(No output)



Observation:

A new file named demo.txt was created.

\----------------------------------------------------

**Step 8:** Create Another File

Command

echo "Git Hands-on Lab" > lab.txt

Output

(No output)

Observation



Another file named lab.txt was created.

\----------------------------------------------------

**Step 9:** Verify Files

Command

ls

Output

demo.txt

lab.txt

Observation



Both files are present in the repository.

\----------------------------------------------------



**Step 10:** Check Repository Status

Command

git status

Output

On branch GitNewBranch



Untracked files:

demo.txt

lab.txt

Observation



Git detected two new untracked files.

\----------------------------------------------------

**Step 11:** Add Files to Staging Area

Command

git add .

Output

(No output)

Observation



All newly created files were added to the staging area.

\----------------------------------------------------

**Step 12:** Verify Staged Files

Command

git status

Output

Changes to be committed:



new file: demo.txt

new file: lab.txt

Observation



The files are successfully staged for commit.

\----------------------------------------------------

**Step 13:** Commit the Changes

Command

git commit -m "Added demo files in GitNewBranch"

Output

\[GitNewBranch abc1234] Added demo files in GitNewBranch



2 files changed

create mode 100644 demo.txt

create mode 100644 lab.txt

Observation



The staged files were committed successfully.

\----------------------------------------------------

**Step 14:** Check Git Status

Command

git status

Output

On branch GitNewBranch



nothing to commit, working tree clean

Observation



The repository is clean and there are no pending changes.

\----------------------------------------------------

**Step 15:** Switch Back to Master Branch

Command

git checkout master

Output

Switched to branch 'master'

Observation



Successfully switched back to the master branch.

\----------------------------------------------------

**Step 16:** Compare Master and GitNewBranch

Command

git diff master GitNewBranch

Output

diff --git a/demo.txt b/demo.txt

new file mode 100644



This file is created in GitNewBranch.



diff --git a/lab.txt b/lab.txt

new file mode 100644



Git Hands-on Lab

Observation



The command displayed the differences between the master branch and GitNewBranch.

\----------------------------------------------------

**Step 17:** View Differences Using P4Merge

Command

git difftool master GitNewBranch

Output

Launching P4Merge...

Observation



The visual comparison between the master branch and GitNewBranch was displayed using P4Merge.

\----------------------------------------------------

**Step 18:** Merge GitNewBranch into Master

Command

git merge GitNewBranch

Output

Updating 1a2b3c4..abc1234



Fast-forward



demo.txt



lab.txt

Observation



The changes from GitNewBranch were merged successfully into the master branch.

\----------------------------------------------------

**Step 19:** View Commit History

Command

git log --oneline --graph --decorate

Output

\* abc1234 (HEAD -> master) Added demo files in GitNewBranch



\* 1a2b3c4 Initial Commit

Observation



The commit history displays the merged commit and branch graph.

\----------------------------------------------------

**Step 20:** Delete the Merged Branch

Command

git branch -d GitNewBranch

Output

Deleted branch GitNewBranch (was abc1234).

Observation



The merged branch GitNewBranch was deleted successfully.

\----------------------------------------------------

**Step 21:** Verify Remaining Branches

Command

git branch

Output

\* master

Observation



Only the master branch remains.

\----------------------------------------------------

**Step 22:** Check Final Repository Status

Command

git status

Output

On branch master



nothing to commit, working tree clean
